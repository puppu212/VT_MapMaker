# VT MapMaker

VT本体付属の `MapMaker 1.10` の `.map` 形式と標準チップを扱える、ブラウザ版マップエディタです。

## 起動

ブラウザのセキュリティ制限により、HTMLファイルの直開きではなくローカルサーバーを使います。

```sh
cd mapmaker-web
python3 -m http.server 4173
```

その後、`http://localhost:4173` を開いてください。

## 実装済み

- Java版 `.map` ファイルの読込・保存
- 標準の地形・建物チップ
- PNGチップの一時追加
- 選択中の地形／建物タブへ任意名フォルダを読み込む素材追加
- `field`／`object`フォルダからのJava版素材セット一括読込
- 個別PNGと`field.dat + fielddata.dat`／`object.dat + objectdata.dat`の自動分類
- 追加PNG・素材セットのブラウザ内永続保存と自動復元
- 編集中マップの自動保存、最大10件のバックアップ
- 起動時の前回作業復元
- ペン、消去、地形の塗りつぶし
- 1～10マスのペン太さ
- 建物の衝突判定と重ね合わせ
- 範囲選択、コピー、貼り付け
- コピー内容の90度回転、左右・上下反転
- コピー範囲の`.map`保存と、別`.map`のコピー素材読込
- コピー内容の半透明プレビューとドラッグ配置
- 上下左右を個別指定したマップの拡張・削減
- 配置記号を保護した消去・貼り付け
- 建物を境界線として扱う塗りつぶし
- ユニット名と `@`、`@@`、`##`、`@ESC@` などの配置
- 配置記号のスポイトと陣形名表示
- `##`の3×3、`@ESC@`の3×2表示
- 建物・配置記号の一時非表示
- Spaceドラッグまたは移動ツールによるマップ移動
- Java版と同じ32px基準の座標表示
- Undo / Redo
- ズーム、グリッド、建物・配置記号の表示切替
- 10～255マスのマップ作成とサイズ変更
- 使用中のツールに応じて設定を切り替える編集中心UI
- 折りたたみ式の表示・マップ・バックアップ設定
- タブレット／小画面向けの開閉式ツール・素材パネル
- PWAインストールとオフライン起動

## 操作

- 左ドラッグ: 選択中のツールを使用
- 右ドラッグ: 消去
- マウスホイール: ポインタ位置を中心にズーム
- Macトラックパッドのピンチ: ポインタ位置を中心にズーム
- トラックパッドの2本指スワイプ: マップをスクロール
- `Space`＋ドラッグ: マップを移動
- `P` / `E` / `F` / `U`: ペン / 消去 / 塗り / 配置
- `S` / `V`: 範囲選択 / 貼り付け
- `I` / `H`: 配置記号の設定取得 / マップ移動
- `Ctrl/Cmd + Z`: Undo
- `Ctrl/Cmd + Shift + Z` または `Ctrl/Cmd + Y`: Redo

画面が狭い場合は、上部左のメニューボタンでツール、右の素材ボタンで素材パレットを開閉できます。ツールまたは素材を選ぶとマップ画面へ自動的に戻ります。

追加したPNGと素材セットはブラウザ内へ保存され、次回起動時に自動復元されます。保存される `.map` にはチップ名のみが記録されるため、ゲーム側でも同名PNGを所定のフォルダへ配置してください。

パレットの「フィールドフォルダを読み込む」「オブジェクトフォルダを読み込む」では、フォルダ名に関係なく、その中のPNGを選択中の種類として追加します。

「素材を管理…」の一括読込では、Java版と同じ`field`／`object`構成の親フォルダ、または各フォルダ単独を選べます。個別PNGと`.dat`内に同名チップがある場合は、個別PNGが優先されます。

## PWA

GitHub PagesなどのHTTPS環境で開くと、対応ブラウザの「インストール」または「ホーム画面に追加」からアプリとして追加できます。Web版とPWA版は同じURL・同じUIです。

インストール可能な環境では、左側の「バックアップ」内に「アプリをインストール」ボタンが表示されます。Chrome／Edgeではブラウザの確認画面が開き、Safariでは端末に応じた追加手順を案内します。インストール済みの場合、ボタンは表示されません。

初回のオンライン起動時にアプリ本体と標準素材を保存し、以降はオフラインでも起動できます。利用者が追加した素材と自動バックアップは、従来どおりブラウザのIndexedDBへ保存されます。

Service Workerの事前キャッシュ内容を変更した場合は、`sw.js`の`CACHE_NAME`も更新してください。

## 開発

主な責務は次のモジュールへ分割しています。

- `app.js`: 起動、編集フロー、Canvas描画
- `app-utils.js`: 座標計算、表示文言、コピー範囲変換
- `ui-controller.js`: ツール表示とレスポンシブパネル制御
- `backup-manager.js`: 自動保存とバックアップ状態
- `materials.js`／`material-loader.js`: 素材分類とインポート
- `pwa.js`／`sw.js`: Service Worker登録とオフラインキャッシュ
- `formats.js`／`model.js`: `.map`形式と編集モデル

回帰テストは次のコマンドで実行できます。

```sh
for test in tests/*.mjs; do node "$test"; done
```
